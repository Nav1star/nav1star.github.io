IMPLEMENTION PROJECT GUIDELINES

All new Implementation Project requests will be submitted using the on-line SharePoint form. Below you are the instructions and guidelines for submitting a new Project request. These guidelines will help us assign resources in a more accurate and efficient manner. When SharePoint is unavailable, please complete the PDF Project Request Form and send to imprequests@cccis.com. Once SharePoint issues have been resolved you will also have to submit your request via the on-line SharePoint form. Submitting your requests via the SharePoint site will provide all internal stakeholders with neccessary project updates.

Before you submit a new project request you must have the following:

  • Signed contract has been submitted and/or appropriate approvals have been acquired.
  • Completed workflow documentation which includes current and proposed client workflow.
  • If your project requires Configurable Workflow (CWF) and/or Predictive Analytics Method of Inspection (PA-MOI) rules, you must attach completed rules requirements with your request.
  • The project requestor and/or Account Team (which includes the CSM or S&C) will be responsbile for securing addtional resources (ex. Interface and/or Product Team) as it relates to the products being implemented.
Note: Submitting a project request with one of the above items missing and/or incomplete will result in a delay of assigning resources until such gaps are met. New project requests are reviewed and assigned every Monday, Wednesday, and Friday at 11a.m. CT.
